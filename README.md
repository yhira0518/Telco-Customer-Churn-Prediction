## プロジェクト概要
本プロジェクトでは、通信会社の顧客データ（IBM Telco Customer Churnデータセット）を使用し、顧客がサービスを解約（チャーン）するかどうかを予測する二値分類モデルを構築しました。

単一のモデルを構築するだけでなく、代表的な5つの機械学習アルゴリズム（決定木、ランダムフォレスト、LightGBM、SVM、k-NN）を実装・チューニングし、それぞれの**予測精度、特徴量への感度、およびビジネスへの解釈性**を比較検証しています。

## データセット
* **ソース:** IBM Telco Customer Churn Dataset
* **特徴量:** 20変数（顧客の基本情報、契約内容、月額料金、総支払額など）
* **目的変数:** `Churn` (Yes: 1, No: 0)

## 評価指標と最終結果
解約予測のような「少数派（解約者）を正確に見つけ出すタスク」において最も重要な指標である **ROC-AUC** をメインの評価指標とし、各モデルのハイパーパラメータ・チューニング（GridSearchCV）を行いました。

| 順位 | モデル | ROC-AUC | 正解率 (Accuracy) | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **LightGBM** | **0.8642** | 0.8141 | 高速かつ最高精度。浅い木(depth=3)で汎化性能を確保 |
| **2** | **Random Forest** | **0.8642** | 0.8048 | LightGBMと同等のAUC。強力なアンサンブル学習 |
| **3** | Decision Tree | 0.8480 | 0.8077 | 単一の木としては健闘。過学習を防ぐためdepth=5に制限 |
| **4** | k-NN (k近傍法) | 0.8436 | 0.7920 | ※上位10特徴量に厳選し、標準化を実施 |
| **5** | SVM | 0.8386 | 0.8041 | ※上位10特徴量に厳選し、標準化を実施 (線形カーネル) |

## ビジネス・インサイト（特徴量重要度からの考察）
決定木ベースのモデル（LightGBM / Random Forest）が算出した特徴量重要度（Feature Importance）から、以下のビジネス上の知見が得られました。

1. **契約期間（`tenure`）が最大の離脱要因:** 契約初期の顧客ほど解約リスクが極めて高い。オンボーディング（導入支援）の手厚いサポートが急務。
2. **価格への敏感さ（`TotalCharges`, `MonthlyCharges`）:** 支払額の高さは明確な解約トリガーとなっている。
3. **光ファイバー契約（`InternetService_Fiber optic`）の罠:** 高速回線であるにもかかわらず解約の強い要因になっている。競合他社より割高であるか、通信品質に不満がある可能性が高く、詳細な調査が必要。
4. **契約形態（`Contract_Two year`）の保持力:** 2年契約を結んでいる顧客は解約リスクが有意に低い。割引キャンペーン等を用いてでも、長期契約へ誘導する施策が有効。

## 技術的な考察（Technical Discussion）

### 1. テーブルデータにおける「決定木ベース」の優位性
本検証において、カテゴリ変数と連続変数が混在するテーブルデータでは、決定木ベースのアンサンブル手法（LightGBM, Random Forest）が高いパフォーマンスを発揮しました。これらはデータのスケールに依存せず、内部で自動的に有用な特徴量を選別（ノイズを無視）できる強みがあることが確認できました。

### 2. 距離ベースモデル（SVM, k-NN）の限界
距離を計算のベースとするSVMやk-NNに対しては、すべての特徴量をそのまま投入すると「次元の呪い」により精度が著しく低下します。
本プロジェクトでは、ランダムフォレストの重要度を用いて**「上位10個の特徴量への厳選」**および**「StandardScalerによる標準化」**という前処理を行ってから学習させました。しかし、フェアマッチを組んだ状態でも決定木ベースのアンサンブルモデル（AUC 0.864）には及ばず、実務におけるLightGBMの手軽さと堅牢性が浮き彫りになる結果となりました。
